# Yumichi OS

🌐 **公開URL**: https://ymakinose618-rgb.github.io/yumichi-brain-os/

牧ノ瀬由美の思考構造を3D空間で可視化する個人OS。

タスク管理ツールではない。
構造設計者として、自分の頭の中の「領域」「観察」「違和感」「自動化候補」がどう配置され、どう繋がり、どこに構造負荷が溜まっているかを観るためのインフラ。

## 設計の核

- **中央コア＝軸**：「構造で考える」「違和感を観察する」「繰り返しは自動化する」「軸で人を見る」「理念で繋がる」「AI を前提に集まり方を設計する」が巡回表示
- **領域（旧フォルダ）**：思考のテーマ単位。仕事 / DX・AI / SNS・広報 / 子ども / 今月のタスク / 人間関係 / 副業・起業 / 感情ログ / アイデア保管庫 / 未処理タスク
- **領域間の関係線**：領域は独立しない。構造的に影響しあう関係を線で接続
- **観察の4類型**：観察 / 構造仮説 / 違和感 / 自動化候補
- **違和感シグナル**：違和感が溜まった領域は赤いハローが脈動する
- **構造負荷モード**：ONで領域が動的に揺れ、観察カードが3D空間に浮遊。自動化候補は中央コア寄りにドリフトする（システム化レイヤーへ流れる）

## ローカルで起動

```
node .claude/server.js
```

`http://localhost:8765` を開く。Python派なら `python -m http.server 8000` でも可。

## 操作

- ドラッグ ＝ 空間を回転
- ホイール / ピンチ ＝ ズーム
- 領域クリック ＝ パネルを開いて観察を記録
- 観察追加時は4類型から選択（観察 / 構造仮説 / 違和感 / 自動化候補）
- 構造負荷モード ＝ 違和感の分布と自動化候補の流れを可視化

## 保存

全データは `localStorage` (KEY: `yumichi-brain-os-v1`) に保存。

## デプロイ

ビルド不要の純静的サイト。GitHub Pages / Cloudflare Pages にそのまま置ける。

## ファイル構成

```
yumichi-brain-os/
├── index.html
├── styles.css
├── README.md
├── .claude/
│   ├── launch.json
│   └── server.js
└── src/
    ├── main.js      シーン・カメラ・raycaster・状態統合
    ├── core.js      中央コア（軸の巡回表示）
    ├── folders.js   領域・関係線・違和感シグナル・構造負荷アニメ
    ├── ui.js        パネル・観察CRUD・4類型ピル
    ├── storage.js   localStorage
    └── data.js      初期領域 / EDGES / NOTE_TYPES / AXIS_STATEMENTS
```

## 拡張の方向

- 軸（AXIS_STATEMENTS）を編集して、自分の信条を書き換える
- EDGES を編集して、領域間の関係性を自分の構造観に合わせる
- 自動化候補が一定数溜まった領域は「システム化候補」として中央コアに集約表示する（将来）
- 領域に「成熟度」「再現性スコア」を追加し、構造健全度ダッシュボード化（将来）
